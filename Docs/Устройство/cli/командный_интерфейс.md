# CLI и основные команды

## Роль CLI в продукте

`agentctl` - основной интерфейс управления системой. Через него пользователь инициализирует проект, подключает правила и prompt templates, создает задачи, запускает агентов, отвечает на уточняющие вопросы, смотрит результаты и принимает решения по review.

CLI важен не только как оболочка для команд. Он задает рабочую модель продукта:

- задача оформляется как команда и файл спецификации;
- сценарий поведения агента оформляется как подключаемый prompt template;
- выполнение запускается явно;
- результаты доступны через просмотр артефактов;
- статусы и история читаются как часть инженерного процесса.

## Базовые группы команд

### Инициализация проекта

```bash
agentctl init
```

Команда создает служебную структуру `.agentctl/`, в которой будут жить конфигурации, правила, шаблоны, задачи и результаты запусков.

### Подключение guidelines

```bash
agentctl guidelines add ./docs/backend-guidelines.md
agentctl guidelines add ./docs/ddd-rules.md
agentctl guidelines add ./docs/testing-policy.md
```

Guidelines добавляются в проект как нормативные документы, которые затем можно прикреплять к задачам.

### Подключение prompt templates

```bash
agentctl prompt-templates add ./prompts/clarify-then-implement.yml
agentctl prompt-templates list
agentctl prompt-templates show clarify-then-implement
```

Prompt templates добавляют в систему переиспользуемые сценарии поведения агента. В отличие от task templates они не описывают поля задачи, а задают режим исполнения: например обязательные вопросы перед реализацией или необходимость сначала показать план.

### Создание задачи

```bash
agentctl task create \
  --title "Implement refresh token repository" \
  --goal "Add repository for refresh tokens in auth module" \
  --agent claude \
  --prompt-template clarify-then-implement \
  --allow "src/Auth/**" \
  --forbid "src/Billing/**" \
  --guide backend-guidelines \
  --guide ddd-rules
```

На этом этапе пользователь описывает цель задачи, выбирает агента, задает scope, прикрепляет правила проекта и сценарий поведения агента.

### Добавление prompt template в существующую задачу

```bash
agentctl task update TASK-001 --prompt-template clarify-then-implement
```

Это позволяет менять режим работы без пересоздания задачи. У одной задачи может быть несколько prompt templates, если они совместимы по сценарию.

### Запуск задачи

```bash
agentctl task run TASK-001
```

Во время запуска система:

- собирает контекст;
- готовит execution prompt;
- применяет prompt templates;
- поднимает изолированную среду;
- передает задачу агенту;
- сохраняет артефакты;
- запускает validation.

Если у задачи подключен двухэтапный шаблон вроде `clarify-then-implement`, первый запуск не приводит к немедленной реализации. Сначала агент анализирует постановку и задает уточняющие вопросы.

### Работа с уточняющими вопросами

```bash
agentctl task run TASK-001
agentctl task questions TASK-001
agentctl task answer TASK-001 --file answers.yml
agentctl task resume TASK-001
```

Такой сценарий делает выполнение двухэтапным:

1. Агент получает задачу и формирует только вопросы.
2. Пользователь отвечает через CLI.
3. Задача возобновляется и переходит к реализации с учетом ответов.

### Просмотр состояния очереди

```bash
agentctl task list
```

Команда показывает текущие статусы задач, например `draft`, `queued`, `clarifying`, `awaiting_answers`, `running`, `review` или `completed`.

### Просмотр результатов

```bash
agentctl result show TASK-001
agentctl result diff TASK-001
```

Через эти команды пользователь получает summary, список измененных файлов, patch, лог выполнения, результаты validation и отмеченные риски.

### Решения по review

```bash
agentctl task accept TASK-001
agentctl task reject TASK-001 --reason "Touched forbidden paths"
agentctl task rerun TASK-001
agentctl task route TASK-001 --agent codex
```

CLI поддерживает не только запуск, но и управление дальнейшей судьбой задачи: принятие, отклонение, повторное выполнение и перенаправление другому агенту.

## Практический сценарий через CLI

Типовой поток работы через консоль выглядит так:

1. Инициализировать агентную среду в репозитории.
2. Подключить guidelines, task templates и prompt templates.
3. Создать задачу с ограничениями, агентом и нужным сценарием поведения.
4. Запустить задачу.
5. Если шаблон требует clarification stage, ответить на вопросы и возобновить выполнение.
6. Дождаться completion или review status.
7. Просмотреть diff и validation.
8. Принять, отклонить или повторно маршрутизировать результат.

## Почему CLI удобен именно для этого продукта

Для Agent-CLI консольный формат естественен, потому что он хорошо сочетается с:

- git и worktree;
- файлами `.yml` и `.md`;
- bash-командами для validation;
- автоматизацией и CI/CD;
- параллельным запуском независимых задач.
